# Recap Format Guide

## Slack Formatting Rules

- Use bold for repo names
- Use bullet points for individual PRs
- Keep the highlights section to 1-2 sentences max
- Include the PR author in parentheses
- Link to the PR if possible

## Category Prefixes

Determine category from the PR title prefix (conventional commits):
- `feat:` or `feature:` = Feature
- `fix:` or `bugfix:` = Fix
- `chore:` = Chore
- `docs:` = Docs
- `refactor:` = Refactor
- No prefix = categorize based on the PR body content
